# Синхронизация с Flash-bot

## Быстрая синхронизация изменений

Вместо ручного копирования файлов используйте скрипт синхронизации:

```bash
python3 sync_to_flash_bot.py
```

### Что делает скрипт:

1. **Определяет измененные файлы** - находит файлы, которые были изменены в текущем репозитории
2. **Фильтрует файлы** - копирует только те файлы, которые нужно синхронизировать (Python модули, конфигурации, workflows)
3. **Копирует файлы** - переносит изменения в Flash-bot
4. **Создает коммит** - автоматически делает коммит в Flash-bot

### Синхронизация всех файлов из списка

Если нужно синхронизировать все файлы из списка (не только измененные):

```bash
python3 sync_to_flash_bot.py --all
```

### Файлы, которые синхронизируются:

- Все Python модули (`.py` файлы)
- GitHub Actions workflows (`.yml` файлы в `.github/workflows/`)
- Конфигурационные файлы (`requirements-github.txt`, `env.example`)

### Файлы, которые НЕ синхронизируются:

- `.env` - файлы с секретами
- `.git` - служебные файлы git
- `__pycache__` - кэш Python
- `*.md` - документация (может отличаться)
- `.vscode` - настройки IDE

### После синхронизации:

Скрипт создаст коммит, но не отправит изменения на GitHub. Для отправки:

```bash
cd /Users/y/Downloads/Flash-bot
git push
```

## Альтернативный способ: Git Remote

Можно настроить Flash-bot для получения изменений напрямую из Basketball-Team-Bot:

```bash
cd /Users/y/Downloads/Flash-bot
git remote add source ../Basketball-Team-Bot
git fetch source
git merge source/master --no-commit
# Выберите нужные файлы
git commit -m "Обновление из Basketball-Team-Bot"
```

Но скрипт `sync_to_flash_bot.py` более удобен, так как автоматически фильтрует файлы.
